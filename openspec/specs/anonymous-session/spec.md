## Purpose

익명 사용자가 별도 로그인이나 서버 저장 사용자 계정 없이 브라우저 세션 기반 UUID와 닉네임으로 채팅에 참여할 수 있게 한다.

## Requirements

### Requirement: 세션 생성 및 닉네임 부여
시스템은 사이트 최초 접속 시 브라우저 sessionStorage에 UUID와 익명 닉네임을 생성하여 저장해야 한다(SHALL). 닉네임은 `익명` + 두 자리 난수(03~99) 형식이어야 한다(MUST). 새 탭/창을 열거나 탭을 닫고 재접속하면 새로운 세션과 닉네임이 부여되어야 한다(SHALL).

#### Scenario: 최초 접속 시 세션 생성
- **WHEN** 사용자가 처음 사이트에 접속한다
- **THEN** sessionStorage에 고유 UUID와 닉네임(예: 익명42)이 저장된다

#### Scenario: 같은 탭에서 새로고침 시 세션 유지
- **WHEN** 사용자가 같은 탭에서 페이지를 새로고침한다
- **THEN** 기존 sessionStorage의 UUID와 닉네임이 유지된다

#### Scenario: 탭을 닫고 재접속 시 새 세션 부여
- **WHEN** 사용자가 탭을 닫고 새 탭으로 사이트에 접속한다
- **THEN** 새로운 UUID와 다른 닉네임이 부여된다

### Requirement: 닉네임 화면 표시
시스템은 현재 세션의 닉네임을 채팅 UI에 표시해야 한다(SHALL). 내가 보낸 메시지에는 내 닉네임이 표시되어야 한다(MUST).

#### Scenario: 내 닉네임 확인
- **WHEN** 사용자가 채팅방에 입장한다
- **THEN** 현재 자신의 익명 닉네임이 UI에 표시된다

#### Scenario: 내 메시지에 닉네임 표시
- **WHEN** 사용자가 메시지를 전송한다
- **THEN** 해당 메시지에 자신의 닉네임이 함께 표시된다
