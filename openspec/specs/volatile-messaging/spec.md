## Purpose

메시지를 짧은 TTL로 저장하고 realtime event stream으로 전달해 휘발성 채팅 경험을 제공한다.

## Requirements

### Requirement: 메시지 전송
시스템은 사용자가 채팅방에서 텍스트 메시지를 전송할 수 있게 해야 한다(SHALL). 메시지는 최대 200자여야 하며(MUST), 서버에 저장되는 즉시 TTL이 설정되어야 한다(MUST).

#### Scenario: 메시지 전송 성공
- **WHEN** 사용자가 텍스트를 입력하고 전송한다
- **THEN** 메시지가 채팅방에 표시되고 TTL 타이머가 시작된다

#### Scenario: 빈 메시지 전송 방지
- **WHEN** 사용자가 빈 메시지 또는 공백만 있는 메시지를 전송 시도한다
- **THEN** 메시지가 전송되지 않는다

#### Scenario: 200자 초과 입력 방지
- **WHEN** 사용자가 200자를 초과하는 텍스트를 입력한다
- **THEN** 200자 이후 입력이 차단되며 현재 글자수가 표시된다

### Requirement: 메시지 자동 소멸 (TTL)
시스템은 전송된 메시지를 설정된 시간(기본 10초) 이후 화면에서 사라지게 하고 서버에서도 삭제해야 한다(SHALL). 비공개 방은 방 생성자가 메시지 유지 시간을 설정할 수 있어야 한다(MUST).

#### Scenario: 메시지 자동 소멸
- **WHEN** 메시지가 전송된 후 설정된 TTL(기본 10초)이 경과한다
- **THEN** 해당 메시지가 화면에서 사라지고 Redis에서 삭제된다

#### Scenario: 비공개 방 메시지 유지 시간 설정
- **WHEN** 비공개 방 생성자가 메시지 유지 시간을 선택한다 (5초, 10초, 30초)
- **THEN** 해당 방의 모든 메시지는 선택된 시간 후 소멸된다

### Requirement: Realtime event stream으로 메시지 수신
시스템은 3초 반복 폴링 대신 realtime event stream으로 새 메시지를 전달해야 한다(SHALL). 클라이언트는 방 입장 시 현재 TTL이 남아있는 메시지 snapshot을 1회 조회하고, 이후 새 메시지는 event stream으로 수신해야 한다(MUST).

#### Scenario: 새 메시지 즉시 수신
- **WHEN** 다른 사용자가 메시지를 전송한다
- **THEN** 해당 메시지가 realtime event를 통해 내 화면에 나타난다

#### Scenario: 이미 소멸된 메시지 미표시
- **WHEN** 방 입장 snapshot 또는 realtime 갱신 시 TTL이 만료된 메시지가 있다
- **THEN** 해당 메시지는 화면에 표시되지 않는다

### Requirement: 스프레드시트 위장 UI
시스템은 채팅 화면을 스프레드시트(엑셀) 배경 위에 메시지 말풍선이 떠다니는 형태로 표시해야 한다(SHALL). 회사에서 일하는 척 할 수 있는 외관을 제공해야 한다(MUST).

#### Scenario: 스프레드시트 배경 표시
- **WHEN** 사용자가 채팅방에 입장한다
- **THEN** 배경은 스프레드시트처럼 보이며 그 위에 메시지 말풍선이 표시된다

#### Scenario: 메시지 입력창 하단 표시
- **WHEN** 사용자가 채팅방에 입장한다
- **THEN** 하단에 메시지 입력창이 있으며 "휘발성 메시지 · 저장되지 않음" 안내가 표시된다
